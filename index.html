One scanner.
Three attack surfaces.

Small enough to run on every commit. Sharp enough to stop the issues that matter before they become incidents.

01 // secrets[ enabled ]

credentials in code

Regex and Shannon entropy detectors catch AWS, GCP, Slack, GitHub, Stripe, JWTs, private keys, and generic API keys.

--git-history
02 // dependencies[ enabled ]

known vulnerabilities

Parse Go, npm, and Python manifests. Query OSV.dev with a local TTL cache and report only packages your code imports.

--format sarif
03 // config[ enabled ]

unsafe defaults

Find exposed .env files, debug flags, default credentials, weak permissions, missing headers, and public Terraform exposure.

--fail-on high

Security feedback
at shell speed.

No dashboard to configure. No agent to maintain. Run locally, run in CI, or inspect only the files changed in a pull request.

[01]

PR-aware by default

--git-diff origin/main keeps feedback focused on what changed.

[02]

Built for the pipeline

Fail on Critical or High. Export table, JSON, or SARIF for GitHub Code Scanning.

[03]

Actionable findings

Every result includes the file, rule, severity, snippet, and a practical next step.

[04]

Respectful of your repo

Honors .gitignore and .sentinelignore. Local cache, no source code leaves your machine.

Readable output.
Useful signal.

Built for humans in a terminal first, with machine-readable output when your pipeline needs it.
